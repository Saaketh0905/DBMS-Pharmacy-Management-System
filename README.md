# DBMS-Pharmacy-Management-System

This project implements a relational database for a pharmacy management system.
It focuses on schema design, data integrity, and procedural logic using SQL and PL/SQL.

The database models core entities such as pharmacies, medicines, doctors, patients,
prescriptions, and supplier relationships, and supports common operational queries
and procedures.

---

## Features

- Well-normalized relational schema
- Primary key and foreign key constraints
- Business rules enforced using CHECK constraints
- Stored procedures and functions for core operations
- Sample queries demonstrating joins and aggregations
- Modular SQL scripts for easy setup and execution

---

## Tech Stack

- **Database**: Oracle SQL  
- **Procedural Language**: PL/SQL  
- **Tools**: Oracle SQL Developer / SQL\*Plus

---

## Notes

-All data used in this project is dummy data created solely for demonstration and learning purposes.
-No real personal, medical, or commercial data is included.
-SQL scripts use relative paths to ensure portability across systems.
